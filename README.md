# Task Tracker

## Описание проекта

Task Tracker — это веб-приложение для управления задачами и проектами, построенное на Django с использованием Django Rest Framework (DRF). Приложение позволяет создавать доски (Kanban-style), организовывать задачи в колонки, управлять участниками и правами доступа. Фронтенд реализован с помощью встроенных Django-шаблонов с инлайновым CSS и JavaScript для интерактивности.

Проект предназначен для командной работы, где пользователи могут создавать доски, назначать задачи, общаться в чате и отслеживать прогресс.

## Основной функционал

### Доски (Boards)
- Создание, редактирование и удаление досок.
- Настройка приватности: публичные (доступны всем) или приватные (только участники).
- Управление участниками: добавление пользователей с ролями (админ, редактор, зритель).
- Настройки прав для участников: просмотр всех задач, редактирование колонок.
- Организационная структура команды и страница 404 для несуществующих страниц.

### Задачи (Tasks)
- Создание, редактирование, удаление задач.
- Назначение исполнителей.
- Комментарии к задачам с упоминаниями пользователей.
- Архивация задач.
- Фильтры: "Мои задачи", "Все задачи" (если есть права), фильтр по пользователю.

### Колонки (Columns)
- Создание, редактирование названий, удаление колонок.
- Перемещение колонок (drag & drop).
- Изменение порядка колонок.

### Участники и права
- Добавление/удаление участников доски.
- Настройка индивидуальных прав: просмотр всех задач, редактирование колонок.
- Роли участников: админ (полный доступ), редактор (редактирование), зритель (просмотр).

### Чат доски
- Общение в реальном времени по доске.
- Ответы на сообщения.
- Уведомления о новых сообщениях.

### Уведомления
- Автоматические уведомления о назначении задач, комментариях и т.д.
- Просмотр и управление уведомлениями.

### Статистика и дашборды
- Личная статистика задач.
- Общий дашборд по всем доскам (для админов и ГС).

### Настройки системы
- Изменение названия системы (для админов и ГС).
- Настройка фона и цветов интерфейса.

## Ролевая модель

### Глобальные роли пользователей
Пользователи имеют роли, определяющие базовые права в системе:

- **admin**: Полный доступ ко всем функциям, включая управление пользователями, досками и настройками системы.
- **chief**: Высокий уровень доступа, аналогично admin, но без некоторых глобальных прав.
- **lead**: Руководитель, может управлять досками и задачами.
- **specialist**: Специалист, ограниченный доступ (видит только свои задачи и задачи подчинённых).
- **senior**: Старший специалист, аналогично specialist.
- **junior**: Младший специалист, минимальные права.

Аббревиатуры ролей:
- **мс** — младший специалист (junior)
- **спец** — специалист (specialist)
- **старший специалист** / senior
- **вс** — ведущий специалист / lead
- **гс** — главный специалист / chief

### Права на досках
Для участников доски (BoardMembership) есть дополнительные права:

- **view_all_tasks**: Может видеть все задачи на доске (включая чужие). Без этого права доступен только фильтр "Мои задачи".
- **edit_columns**: Может редактировать колонки (создавать, изменять названия, удалять, перемещать).
- **reorder_columns**: Может менять порядок колонок (включено в edit_columns).

### Роли участников доски
- **admin**: Полный доступ к доске, включая управление участниками.
- **editor**: Может редактировать доску и задачи.
- **viewer**: Только просмотр.

### Иерархия прав
- Глобальные роли имеют приоритет над правами на досках.
- Создатель доски имеет все права на неё.
- Админы системы имеют все права везде.

## Технологии

- **Backend**: Django 4.x, Django Rest Framework
- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **База данных**: SQLite (для разработки), PostgreSQL (для продакшена)
- **Аутентификация**: JWT (djangorestframework-simplejwt)
- **Дополнительно**: Drag & Drop для задач и колонок, WebSockets для чата (через Django Channels, если настроено)
