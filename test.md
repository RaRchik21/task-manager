# Техническое задание для QA-тестировщика (Task Manager)

## Сводка
- **Проект:** Task Manager – корпоративная система управления задачами с ролевой моделью, досками, канбан‑доской, комментариями, уведомлениями, чатом и оргструктурой.
- **Цель тестирования:** Проверить соответствие функциональных требований, выявить дефекты до передачи в продуктив.
- **Окружение:** Боевой стенд (Render / PythonAnywhere) или локальный сервер разработки.
- **Браузеры:** Chrome, Firefox, Edge (последние версии).
- **Тестовые учётные записи:** Admin, Chief, Lead, Specialist, Senior, Junior (пароли известны).

---

## 1. Авторизация и доступ

### 1.1 Логин
- [ ] Ввод корректных логина/пароля → редирект на `/boards/`
- [ ] Неверные данные → сообщение «Неверный логин или пароль»
- [ ] Пустые поля → ошибка валидации (запрос не отправляется)

### 1.2 Выход
- [ ] Нажатие кнопки «Выйти» → очистка токена, редирект на страницу логина

---

## 2. Страница списка досок (`boards.html`)

### 2.1 Отображение досок по ролям
- [ ] Admin, Chief, Lead – видят все доски (публичные, приватные, где являются участником или создателем)
- [ ] Specialist, Junior, Senior – видят только: публичные с allow_anyone_view=true, доски где участник, доски где создатель

### 2.2 Создание доски
- [ ] Обязательное поле «Название» – пустое значение даёт ошибку
- [ ] Все поля сохраняются: название, описание, приватность, чекбокс allow_anyone_view
- [ ] Участники (редакторы) и зрители (выбор из списка) – добавляются корректно
- [ ] Создатель автоматически получает роль admin на доске
- [ ] После создания доска появляется в списке у создателя и у других согласно правам

### 2.3 Редактирование доски (кнопка «Редактировать»)
- [ ] Кнопка видна только создателю доски или admin системы
- [ ] Возможно изменить: название, описание, приватность, чекбокс, список участников/зрителей
- [ ] После сохранения изменения сразу отображаются в списке

### 2.4 Модалка участников доски (клик по карточке доски)
- [ ] Открывается список участников с именами, ролями, правами
- [ ] Создатель/admin может:
  - [ ] Менять роль (admin, editor, viewer) через select
  - [ ] Удалять участников (кнопка ✕)
  - [ ] Добавлять новых участников (выпадающий список + кнопка «Добавить»)
  - [ ] Назначать дополнительные права: «Видеть все задачи», «Редактировать колонки», «Менять порядок колонок» (чеки)
- [ ] Роли и права сохраняются, интерфейс обновляется без перезагрузки

---

## 3. Канбан-доска (страница `/?board=ID`)

### 3.1 Загрузка доски
- [ ] При переходе с параметром `board` отображаются колонки и задачи текущей доски
- [ ] При попытке открыть доску без доступа (приватная, allow_anyone_view=false, пользователь не участник) – показывается сообщение об ошибке доступа

### 3.2 Колонки
- [ ] Кнопка «+ Добавить колонку» доступна для admin/chief и для участников с правом `edit_columns`
- [ ] Создание колонки (название, цвет) – работает, колонка отображается
- [ ] Редактирование колонки (✎) – доступно тем же, можно изменить название и цвет
- [ ] Удаление колонки (✕) – подтверждение, удаляются все задачи в колонке
- [ ] Перетаскивание колонок (drag & drop) – меняется порядок, доступно при праве `reorder_columns`
- [ ] Сворачивание колонки (если есть) – скрывает задачи, заголовок остаётся

### 3.3 Задачи
- [ ] Создание задачи – кнопка «+ Добавить задачу» в колонке доступна всем участникам доски
- [ ] Поля: название (обяз), описание, приоритет, дедлайн, чекбокс «⭐ В целях ГС» – сохраняются
- [ ] Редактирование задачи (✎) – доступно автору или редактору/админу доски
- [ ] Перемещение задачи между колонками (drag & drop) – работает, задача переходит в новую колонку
- [ ] Архивация задачи (📦) – задача уходит в архив, пропадает с доски
- [ ] Восстановление из архива – задача возвращается в исходную колонку
- [ ] Удаление навсегда из архива – задача исчезает из БД

### 3.4 Фильтрация задач
- [ ] **Мои задачи** – показываются задачи, где пользователь автор или исполнитель
- [ ] **Все задачи** – доступно для admin/chief/lead или если есть право `view_all_tasks` на доске
- [ ] **По пользователю** – выпадающий список подчинённых (зависит от роли), показываются задачи выбранного
- [ ] Поиск по названию / комментарию – работает в реальном времени

---

## 4. Комментарии

### 4.1 Просмотр и создание
- [ ] Вкладка «Комментарии» в модалке задачи – отображает существующие комментарии
- [ ] Поле ввода текста, кнопка «Отправить» – комментарий добавляется
- [ ] @упоминание – при вводе `@` выпадает список пользователей, после отправки упомянутый получает уведомление

### 4.2 Ответы на комментарии
- [ ] Кнопка «Ответить» под каждым комментарием – при клике появляется индикатор ответа с цитатой исходного сообщения
- [ ] Отправленный ответ отображается с отступом и ссылкой на исходный комментарий
- [ ] Клик по цитате – прокрутка к исходному комментарию

### 4.3 Уведомления об упоминаниях
- [ ] При @упоминании – уведомление приходит в колокольчик

---

## 5. Уведомления

### 5.1 Типы и условия
- [ ] **Назначен исполнителем** – появляется, когда пользователя добавили в `assigned_to` задачи
- [ ] **Упомянут в комментарии** – при @упоминании
- [ ] **Предложена задача** – (если реализовано) от нижестоящего вышестоящему
- [ ] **Делегирована задача** – от вышестоящего к исполнителю

### 5.2 Интерфейс
- [ ] Колокольчик в шапке – отображает счётчик непрочитанных
- [ ] Клик – открывает окно уведомлений (модалка / боковая панель)
- [ ] Непрочитанные уведомления выделены (фон, жирный текст)
- [ ] Клик по уведомлению – открывает задачу (переход в нужную доску и модалка задачи), помечает как прочитанное
- [ ] Кнопка «Прочитать все» – помечает все уведомления прочитанными, счётчик обнуляется

### 5.3 Фильтрация уведомлений по доске
- [ ] На странице конкретной доски загружаются только уведомления, относящиеся к этой доске
- [ ] При переключении доски список уведомлений в окне обновляется

---

## 6. Участники доски (вкладка в `index.html`)

(Кнопка «👥 Участники» в шапке основной доски)
- [ ] Открывается боковая панель со списком участников текущей доски
- [ ] Создатель/admin могут:
  - [ ] Менять роли (admin, editor, viewer) через select
  - [ ] Удалять участников (кнопка ✕)
  - [ ] Добавлять участников (выпадающий список + кнопка «Добавить»)
  - [ ] Назначать доп. права (видеть все задачи, редактировать колонки, менять порядок колонок) – чекбоксы
- [ ] Изменения сохраняются и сразу влияют на права доступа

---

## 7. Оргструктура (`orgchart.html`)

- [ ] Доступна только для ролей Specialist и выше (Admin, Chief, Lead, Specialist)
- [ ] Отображает иерархию `supervisor` в виде дерева
- [ ] Скрытые Главные специалисты (`is_hidden=true`) не отображаются в дереве, но попадают в блок «Без руководителя»
- [ ] Создатель/Chief могут:
  - [ ] Изменить руководителя (кнопка «Изменить руководителя») – модалка, выбор только вышестоящих по роли
  - [ ] Скрыть/показать Главного специалиста (кнопка «Скрыть» / «Показать»)

---

## 8. Дашборд (статистика)

- [ ] Кнопка «📊 Статистика» доступна admin/chief/lead
- [ ] Отображает данные **только по текущей доске** (фильтр по board)
- [ ] Карточки: всего задач, просрочено, в целях ГС, в архиве
- [ ] График активности за 7 дней (количество задач по дням)
- [ ] Распределение задач по колонкам (визуальные бары)
- [ ] Распределение по приоритетам (High/Medium/Low)
- [ ] Топ-10 сотрудников по задачам (для ролей admin/chief/lead)

---

## 9. Настройки внешнего вида (кнопка «⚙ Настройки»)

- [ ] Доступна для admin/chief/lead (и для любого с canManage)
- [ ] Изменение темы (готовые, градиент, цвета, прозрачность) сохраняется **только в localStorage** и не синхронизируется с другими пользователями
- [ ] Изменение названия системы – доступно только Главному специалисту/Admin, синхронизируется для всех (запрос на сервер)

---

## 10. Чат доски (кнопка «💬 Чат»)

- [ ] Открывается плавающая панель чата (`#boardChatPanel`)
- [ ] Панель можно перемещать (drag & drop) и изменять размер (resize)
- [ ] Отправка сообщений – поле ввода, кнопка «Отправить», Enter (без Shift) – сообщение отправляется
- [ ] Ответы на сообщения – кнопка «Ответить» под каждым сообщением, индикатор ответа
- [ ] Счётчик новых сообщений – при открытой панели отображается количество новых (не прочитанных) сообщений в бейдже

---

## 11. Негативные и граничные сценарии

### 11.1 Ошибки валидации
- [ ] Создание доски с пустым названием – ошибка, доска не создаётся
- [ ] Отправка пустого комментария – сообщение не отправляется
- [ ] Добавление пользователя в доску, который уже там есть – не добавляется, нет дубля

### 11.2 Доступ и безопасность
- [ ] Удаление колонки с задачами – подтверждение, все задачи удаляются
- [ ] Попытка удалить создателя доски – запрещено (кнопка недоступна или удаление не работает)
- [ ] Попытка переместить колонку без права `reorder_columns` – drag & drop неактивен
- [ ] Попытка открыть приватную доску без доступа – страница с ошибкой «Нет доступа»

### 11.3 Работа с данными
- [ ] @упоминание несуществующего пользователя – не создаётся уведомление, комментарий отправляется без упоминания
- [ ] Восстановление задачи из архива – задача возвращается в ту же колонку, где была
- [ ] Удаление навсегда – задача исчезает, не влияет на другие таблицы (ссылочная целостность)

---

## 12. Обновление в реальном времени (polling)

- [ ] Доска автоматически перезагружается каждые 5 секунд – новые задачи, перемещения отображаются без F5
- [ ] Уведомления подгружаются каждые 15 секунд – счётчик непрочитанных обновляется

---

## 14. Приёмка

Версия считается стабильной и готовой к демонстрации или релизу при выполнении **всех** чекбоксов (кроме возможно помеченных как «не реализовано») и отсутствии критических дефектов (ошибки 500, потеря данных, нерабочие основные функции).
